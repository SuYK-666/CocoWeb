# 风险分析记录

## 模块范围

本次补强模块为 Web 分析任务访问控制，涉及 `webapp.py` 中的任务创建、实时日志流、结果查询与报告下载接口，以及 `web/static/app.js` 中的前端同步逻辑。

## 敏感操作

- 查询分析任务运行日志，日志中可能包含仓库地址、执行路径、错误信息与报告路径。
- 读取 HTML/Markdown/DOCX 报告，报告可能包含目标仓库结构、依赖、代码洞察与安全风险判断。
- 下载最终产物，属于任务结果数据存取行为。

## 主要安全风险

1. 未授权读取任务结果：原实现只依赖 `job_id` 访问 `/api/result/<job_id>`、`/api/stream/<job_id>` 和 `/api/download/<job_id>/<fmt>`。如果链接被浏览器历史、日志或他人转发泄露，第三方可直接读取报告。
2. 任务标识被当作权限凭据：`job_id` 虽然是 UUID，但它同时承担“定位任务”和“访问授权”两种职责，不符合认证授权分层原则。
3. 下载接口缺少独立校验：报告下载接口在检查任务状态前未做访问控制，可能导致报告产物被非创建者获取。

## 安全目标

- 每个任务创建后生成独立访问令牌，将任务定位标识与访问凭据分离。
- 结果查询、实时日志、报告下载均必须校验访问令牌。
- 错误令牌返回 403，缺失令牌返回 401，避免把授权失败伪装成功能异常。
- 使用标准库安全随机数生成令牌，并使用常量时间比较降低时序侧信道风险。
