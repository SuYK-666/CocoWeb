# 项目约束文档

## 背景说明

GitHub Project Analyzer 是一个本地 Web 分析工具。用户提交 API Key、GitHub 仓库地址和分析方向后，系统异步生成报告，并通过 SSE、轮询和下载接口返回任务状态与产物。

## 任务范围

本次只允许在 `成员代码/苏禹宽_GithubProjectAnalyzer` 目录内修改代码和新增文件。功能目标是为 Web 任务增加访问控制，保护任务日志、结果和报告下载。

## 安全约束

- 不把 DeepSeek API Key 或 GitHub Token 写入日志、报告、测试文件或文档样例。
- 不把 `job_id` 作为唯一访问凭据，必须新增独立的任务访问令牌。
- 任务访问令牌必须由安全随机数生成，不能由时间戳、用户名、仓库名拼接而来。
- 校验令牌时使用常量时间比较。
- `/api/result/<job_id>`、`/api/stream/<job_id>`、`/api/download/<job_id>/<fmt>` 三类接口必须统一执行访问控制。
- 前端只能在当前会话中使用任务访问令牌，不写入 localStorage。
- 新增测试必须验证缺失令牌、错误令牌、正确令牌三种情况。

## 禁止行为

- 禁止修改个人目录以外的代码来绕过问题。
- 禁止直接合并 PR。
- 禁止提交真实密钥、个人 Token、运行产生的 `logs/` 或 `data_workspace/`。
- 禁止只写注释或文档而不落实代码逻辑。
